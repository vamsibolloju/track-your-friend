"Track your friend" is very simple mobile tracking browser based application. It uses browser's navigator object to grab the current location of the device for tracking purpose.

Use cases:
Here is the list of usecases fulfilled by this application
  • One can know the current location of one's friends.
  • One can track the location of one's friends
  • One can set the destination for one of one's friend, once that friend reached the destination the user will be notified. If the frind blocks at any location more that 5 minuits with out moving, the user will be notified with the same.
Flow:
Here is the flow for a user to track his/her friends:
  • User should athenticate to this application. New users can create account and authenticate.
  • User will have an option to add other users as friends.
  • User need to request friend to know the location of the friend, once the user approves the request then the user can track his frind's location.

Here is the list of pages with description available in the application.

Login: This is the default page of this application for non authenticated users. This page allows user to authenticate in to this application with username and password. If user is authenticated already and tries to navigate this page, application will forward the request to default page of authenticated users which is track page.

Signup: This page allows new users to create account in to this application. We can navigate to this page by clicking signup link available in login page.

Track: This is the main page of this application. This page is for authenticated users only. If any user tries to navigate this page without authentication, application will forward the request to login page to force the user to authenticate first. If the user is new to this application, that user can create account first by using signup link in the login page. This page shows the list of friends of the user, from there user can select any friend and track the mobile of the friend's location in the map available on top of the list of friends section. The map is visible when only any of the friend is selected by clicking on it. There is a button to navigate to "add a friend" page below the friends list section.

Add a friend: This page allows the user to add any non friend to make friend. A serach input box is available which is used to search non friends list with username of mobile number. Below of the search input a list of non friends of the user is displayed from there we can add the non friend as a friend of the user by clicking add button of the non friend item.

About: This page contains the information about this application, it's use caes, Technologies used for this application, and some references. This page doesn't require any authentication.

This application is developed by implementing client side, server side and database.
Client:
Client application is implemented by using HTML, CSS, Javascript. We choose angular framework to develop this application.
Server:
We implemented two servers, one is for creating rest api and other is for maintaining sockets. RestAPI is implemented with PHP.
Database:
We use MySql database to store users collection and it's related friends.
As we implemented client, server and database applications for this application, we need to deploy all these three applications. We choose almost cheap/free and best hosting services for these applications.
Client:
To deploy client aplications there are many options ( free and best ) available. We choose Surge from them, as it is very easy and efficient to use.
Server:
To deploy server applications ( both PHP and NodeJS applications ) we use heroku .
Database
To deploy MySql database on cloud, we use remote mysql
